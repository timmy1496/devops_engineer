# Здобути практичні навички роботи з BASH.

---

## 1. Автоматичне резервне копіювання директорії

Було створено Bash-скрипт для автоматичного резервного копіювання директорії:

- **Джерело:** `/home/timmy/data`
- **Призначення:** `/home/timmy/backup`

Скрипт копіює вміст директорії збереженням структури та перезаписує попередні копії за потреби.

Для автоматизації запуску було налаштовано **cron job**, який виконує даний скрипт **щодня о 02:00**.

Це дозволяє забезпечити регулярне резервне копіювання без ручного втручання.

---

## 2. Systemd сервіс для перевірки доступності веб-сайту

Було створено Bash-скрипт, який:
- перевіряє доступність заданого веб-сайту (через `curl` або аналогічний інструмент),
- фіксує результат перевірки (доступний / недоступний),
- записує інформацію у лог-файл.

Для автоматичного запуску цього скрипта було:
- створено власний **systemd service**,
- налаштовано його автозапуск,
- перевірено коректну роботу сервісу через `systemctl`.

Таким чином, перевірка доступності веб-сайту відбувається автоматично та централізовано керується systemd.

---

## 3. Моніторинг використання ресурсів системи

Було написано Bash-скрипт для моніторингу основних ресурсів системи, зокрема:
- використання CPU,
- використання оперативної пам’яті,
- використання дискового простору.

Результати виконання скрипта зберігаються у файл для подальшого аналізу. Це дозволяє відслідковувати стан системи та виявляти потенційні проблеми з ресурсами.

---

## 4. Ротація access-логів Nginx

Було налаштовано ротування access-логів Nginx з використанням стандартних інструментів Linux (logrotate).

Налаштування включають:
- періодичну ротацію логів,
- збереження обмеженої кількості архівів,
- стиснення старих лог-файлів,
- коректну роботу Nginx після ротації.

Це дозволяє запобігти неконтрольованому зростанню лог-файлів і перевантаженню дискового простору.
