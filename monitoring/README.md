## Передумови

```bash
# Перевірити наявність інструментів
terraform version   # >= 1.5.0
ansible --version   # >= 2.14
aws --version       # AWS CLI v2

# Налаштувати AWS credentials
aws configure
# або через змінні середовища:
export AWS_ACCESS_KEY_ID="your-key"
export AWS_SECRET_ACCESS_KEY="your-secret"
export AWS_DEFAULT_REGION="eu-central-1"

# SSH ключ
ssh-keygen -t rsa -b 4096 -f ~/.ssh/id_rsa
```

## Швидкий старт

```bash
# 1. Клонувати репозиторій
git clone <repo-url>
cd monitoring-infra

# 2. Налаштувати змінні Terraform
cp terraform/terraform.tfvars.example terraform/terraform.tfvars
# Відредагувати terraform/terraform.tfvars:
#   - allowed_ssh_cidr: твій IP (curl ifconfig.me)/32
#   - public_key_path / private_key_path

# 3. Повне розгортання одною командою
make deploy

# Або покроково:
make init          # terraform init
make plan          # terraform plan
make apply         # terraform apply (створює EC2 + inventory для Ansible)
make ansible-deploy # ansible-playbook (налаштовує всі сервери)
```

## Структура проекту

```
monitoring-infra/
├── Makefile                          # Команди для управління
├── terraform/
│   ├── main.tf                       # VPC, SG, EC2, EIP
│   ├── variables.tf                  # Змінні
│   ├── outputs.tf                    # Outputs
│   ├── terraform.tfvars.example      # Приклад конфігурації
│   └── templates/
│       ├── inventory.tmpl            # Шаблон Ansible inventory
│       └── group_vars.tmpl           # Шаблон group_vars
└── ansible/
    ├── ansible.cfg
    ├── site.yml                      # Main playbook
    ├── inventory/
    │   └── hosts.ini                 # Auto-generated by Terraform
    ├── group_vars/
    │   └── all.yml                   # Auto-generated by Terraform
    └── roles/
        ├── common/                   # Базове налаштування серверів
        ├── monitoring/               # Prometheus + Grafana + Loki
        │   ├── tasks/main.yml
        │   ├── handlers/main.yml
        │   └── templates/
        │       ├── docker-compose.yml.j2
        │       ├── prometheus.yml.j2
        │       ├── loki-config.yml.j2
        │       ├── promtail-config-monitoring.yml.j2
        │       ├── grafana-datasources.yml.j2
        │       ├── grafana-dashboard-provider.yml.j2
        │       └── grafana.env.j2
        └── webserver/                # Nginx + Node Exporter + Promtail
            ├── tasks/main.yml
            ├── handlers/main.yml
            └── templates/
                ├── nginx.conf.j2
                ├── nginx-default.conf.j2
                ├── node-exporter.service.j2
                ├── nginx-exporter.service.j2
                ├── promtail-config-webserver.yml.j2
                └── promtail.service.j2
```

## Корисні команди

```bash
# Перевірити статус сервісів
make verify

# SSH на сервери
$(terraform -chdir=terraform output -raw ssh_monitoring)
$(terraform -chdir=terraform output -raw ssh_webserver)

# Перезапустити тільки monitoring stack
ansible-playbook -i ansible/inventory/hosts.ini ansible/site.yml --tags monitoring

# Перевірити Docker контейнери на monitoring server
ssh ubuntu@<MONITORING_IP> "docker compose -f /opt/monitoring/docker-compose.yml ps"

# Знищити всю інфраструктуру
make destroy
```

## Безпека (для продакшену)

- [ ] Замінити `allowed_ssh_cidr` на конкретний IP замість `0.0.0.0/0`
- [ ] Змінити пароль Grafana в `group_vars/all.yml`
- [ ] Увімкнути HTTPS для Grafana (nginx reverse proxy + Let's Encrypt)
- [ ] Обмежити доступ до портів 9090, 3100, 9100 тільки з VPC
- [ ] Налаштувати Grafana authentication (LDAP/OAuth)
- [ ] Використовувати AWS Secrets Manager для секретів
