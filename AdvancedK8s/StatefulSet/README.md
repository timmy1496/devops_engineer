# Redis StatefulSet у Kubernetes (Minikube)

## Опис роботи
У роботі було розгорнуто Redis-кластер у Kubernetes з використанням StatefulSet. StatefulSet забезпечує стабільні імена pod’ів та постійне зберігання даних за допомогою PersistentVolumeClaim (PVC). Робота виконувалась у середовищі Minikube.

---

## Мета роботи
- Розгорнути Redis-кластер з двома репліками
- Забезпечити стабільні імена pod’ів
- Налаштувати постійне зберігання даних
- Перевірити збереження даних після перезапуску pod’ів

---

## Хід виконання роботи

### 1. PersistentVolumeClaim
Для зберігання даних Redis було використано `volumeClaimTemplates` у StatefulSet. Для кожного pod автоматично створюється окремий PVC, що гарантує збереження даних між перезапусками.

---

### 2. StatefulSet
Створено StatefulSet з двома репліками Redis. Кожен pod має стабільне ім’я (`redis-0`, `redis-1`) і власний том для зберігання даних.

---

### 3. Service
Для взаємодії між pod’ами створено headless Service типу `ClusterIP`, що дозволяє використовувати стабільні DNS-імена (`redis-0.redis`, `redis-1.redis`).
