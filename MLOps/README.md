## Короткий аналіз

У цьому завданні був створений сервіс Customer Support з endpoint `POST /ask`, який використовує LLM для генерації відповідей. Було протестовано два підходи: використання **SaaS API (Gemini)** та **локально запущеної моделі через Ollama**.

SaaS-підхід простіший у реалізації, працює швидше та зазвичай дає якісніші відповіді, оскільки використовує потужну інфраструктуру провайдера. Однак він залежить від зовнішнього сервісу та може вимагати оплати за використання.

Self-hosted модель дозволяє повністю контролювати дані та працювати без інтернету, але на CPU працює повільніше і може давати менш точні відповіді.

Загалом для більшості production-систем зручнішим є **SaaS-підхід**, тоді як **self-hosted** підходить для сценаріїв з підвищеними вимогами до конфіденційності.
